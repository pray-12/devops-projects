# Automated VM Monitoring and Email Alerting System using Ansible

This project demonstrates how to use **Ansible** to monitor multiple virtual machines and send automated reports.

If we have multiple VMs that need to be monitored regularly. The goal is to collect system metrics such as **CPU usage, RAM usage, and disk utilization** from all the machines and generate a consolidated report.

Finally, the report is sent to a specified email address, providing a clear overview of system health across all machines.

This project showcases Ansible’s ability to handle **large-scale automation, remote monitoring, scheduling, and reporting** in a production-like environment.

### Step 5: Script to rename the instances

```bash
#!/bin/bash

# Fetch instance IDs that match Environment=dev and Role=web
instance_ids=$(aws ec2 describe-instances \
  --filters "Name=tag:Environment,Values=dev" "Name=instance-state-name,Values=running" \
  --query 'Reservations[*].Instances[*].InstanceId' \
  --output text)

# Sort instance IDs deterministically
sorted_ids=($(echo "$instance_ids" | tr '\t' '\n' | sort))

# Rename instances sequentially
counter=1
for id in "${sorted_ids[@]}"; do
  name="web-$(printf "%02d" $counter)"
  echo "Tagging $id as $name"
  aws ec2 create-tags --resources "$id" \
    --tags Key=Name,Value="$name"
  ((counter++))
done
```

To run the script, save it as `rename_instances.sh`, give it execute permissions, and run it:

```bash
sudo chmod +x rename_instances.sh
./rename_instances.sh
```

```bash
#!/bin/bash

# Define vars
PEM_FILE="DevOps-Shack.pem"
PUB_KEY=$(cat ~/.ssh/id_rsa.pub)
USER="ubuntu"  # or ec2-user
INVENTORY_FILE="inventory/aws_ec2.yaml"

# Extract hostnames/IPs from dynamic inventory
HOSTS=$(ansible-inventory -i $INVENTORY_FILE --list | jq -r '._meta.hostvars | keys[]')

for HOST in $HOSTS; do
  echo "Injecting key into $HOST"
  ssh -o StrictHostKeyChecking=no -i $PEM_FILE $USER@$HOST "
    mkdir -p ~/.ssh && \
    echo \"$PUB_KEY\" >> ~/.ssh/authorized_keys && \
    chmod 700 ~/.ssh && \
    chmod 600 ~/.ssh/authorized_keys
  "
done
```

### Step8: Collect Metrics and Send Email
`collect_metrics.yaml`

```yaml
- name: Collect VM metrics
  hosts: env_dev
  become: true
  gather_facts: true
  tasks:

    - name: Install sysstat (for mpstat)
      apt:
        name: sysstat # Install sysstat package to get mpstat command for CPU usage
        state: present
      when: ansible_os_family == "Debian"

    - name: Install sysstat (RedHat/CentOS)
      yum:
        name: sysstat
        state: present
      when: ansible_os_family == "RedHat"

    - name: Get CPU usage via mpstat
      shell: "mpstat 1 1 | awk '/Average/ && $NF ~ /[0-9.]+/ {print 100 - $NF}'"
      register: cpu_usage

    - name: Get memory usage
      shell: "free | awk '/Mem/{printf(\"%.2f\", $3/$2 * 100.0)}'"
      register: mem_usage

    - name: Get disk usage
      shell: "df / | awk 'NR==2 {print $5}' | tr -d '%'"
      register: disk_usage

    - name: Set metrics fact
      set_fact:
        vm_metrics:
          hostname: "{{ inventory_hostname }}"
          cpu: "{{ cpu_usage.stdout | float | round(2) }}"
          mem: "{{ mem_usage.stdout | float | round(2) }}"
          disk: "{{ disk_usage.stdout | float | round(2) }}"
```

`send_report.yaml`

```yaml
- name: Send consolidated VM report
  hosts: localhost
  gather_facts: true
  vars:
    collected_metrics: >-
      {{
        hostvars |
        dict2items |
        selectattr('value.vm_metrics', 'defined') |
        map(attribute='value.vm_metrics') |
        list
      }}
    timestamp: "{{ ansible_date_time.date }} {{ ansible_date_time.time }}"
    subject_line: "📊 VM Report – {{ ansible_date_time.date }} {{ ansible_date_time.hour }}:{{ ansible_date_time.minute }}"
  tasks:
    - name: Send animated HTML report via email
      mail:
        host: "{{ smtp_server }}"
        port: "{{ smtp_port }}"
        username: "{{ email_user }}"
        password: "{{ email_pass }}"
        to: "{{ alert_recipient }}"
        subject: "{{ subject_line }}"
        body: "{{ lookup('template', 'templates/report_email_animated.html.j2') }}"
        subtype: html
```

## Conclusion

This project demonstrates a complete **end-to-end DevOps automation solution using Ansible** to monitor multiple virtual machines and generate consolidated health reports. It automates the collection of key system metrics such as **CPU usage, memory usage, and disk utilization** from multiple managed nodes and converts them into a structured, centralized report.

The solution also integrates **SMTP-based email automation**, enabling real-time and scheduled reporting to stakeholders without manual intervention. By using **Ansible dynamic inventory, Jinja2 templating, and scheduled playbooks**, the project ensures scalability, reusability, and consistency across infrastructure.

Overall, this project showcases strong DevOps capabilities in:

- Infrastructure automation using Ansible  
- Multi-node server monitoring  
- Configuration management at scale  
- Automated reporting and email notifications  
- Cloud-based VM orchestration (AWS EC2)

This project can be further enhanced by integrating advanced monitoring tools like **Prometheus, Grafana, or AWS CloudWatch** for real-time visualization and alerting dashboards.
