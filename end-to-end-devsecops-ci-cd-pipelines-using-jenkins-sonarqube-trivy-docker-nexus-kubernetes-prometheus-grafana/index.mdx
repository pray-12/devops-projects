## Step by Step Implementation

### Step 1: Create EC2 instance and attach security group to it.
- Launch 3 instance with OS: 3GB , VM: 24.04 , Instance type: `m7i-flex.large` add security group to it. Also, create a key pair for the instances.

Name them: **Master , slave 1 slave 2**

### Step 2: Setup K8-Cluster on Master Node and slaves.

#### 1. Update System Packages [On Master & Worker Node]

```bash
sudo apt-get update
```

#### 2. Install Docker[On Master & Worker Node]

```bash
sudo apt install docker.io -y
sudo chmod 666 /var/run/docker.sock
```

#### 3. Install Required Dependencies for Kubernetes[On Master & Worker Node]

```bash
sudo apt-get install -y apt-transport-https ca-certificates curl gnupg
sudo mkdir -p -m 755 /etc/apt/keyrings
```

#### 4. Add Kubernetes Repository and GPG Key[On Master & Worker Node]

```bash
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.28/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.28/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list
```

#### 5. Update Package List[On Master & Worker Node]

```bash
sudo apt update
```

#### 6. Install Kubernetes Components[On Master & Worker Node]

```bash
sudo apt install -y kubeadm=1.28.1-1.1 kubelet=1.28.1-1.1 kubectl=1.28.1-1.1
```
** We will get error like this:
```bash
[ERROR FileContent--proc-sys-net-bridge-bridge-nf-call-iptables]:
/proc/sys/net/bridge/bridge-nf-call-iptables does not exist
```
means Kubernetes could not find the Linux bridge networking setting required for container networking.

solution:

```bash
sudo modprobe br_netfilter
ls /proc/sys/net/bridge
sudo sysctl net.bridge.bridge-nf-call-iptables=1
sudo sysctl net.ipv4.ip_forward=1
cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables = 1
net.ipv4.ip_forward = 1
EOF # To make the changes permanent.
sudo sysctl --system # To apply the changes.
```
Then run the kubeadm init command again.

#### 7. Initialize Kubernetes Master Node [On MasterNode]

```bash
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
```
We will get the join command to join the worker nodes to the cluster. Copy it and run it on the worker nodes.

#### 8. Configure Kubernetes Cluster [On MasterNode]

```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

#### 9. Deploy Networking Solution (Calico) [On MasterNode]

```bash
kubectl apply -f https://docs.projectcalico.org/manifests/calico.yaml
```

#### 10. Deploy Ingress Controller (NGINX) [On MasterNode]

```bash
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/controller-v0.49.0/deploy/static/provider/baremetal/deploy.yaml
```

Run the join command on the worker nodes.
```bash
kubectl get nodes
```

#### 11. Install Kubeaudit  for checking the security of the cluster.
```bash
https://github.com/Shopify/kubeaudit/releases # Go to the releases page and download the latest version of kubeaudit.
wget https://github.com/Shopify/kubeaudit/releases/download/v0.22.2/kubeaudit_0.22.2_linux_amd64.tar.gz
tar -xzvf kubeaudit_0.22.2_linux_amd64.tar.gz
sudo mv kubeaudit /usr/local/bin/
kubeaudit all
```

#### 12. Install Trivy for scanning the vulnerabilities in the cluster.
```bash
https://github.com/aquasecurity/trivy/releases
wget https://github.com/aquasecurity/trivy/releases/download/v0.42.0/trivy_0.42.0_linux_amd64.tar.gz
tar -xzvf trivy_0.42.0_linux_amd64.tar.gz
sudo mv trivy /usr/local/bin/
trivy scan --all-packages --exit-code 1
```
