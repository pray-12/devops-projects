# CI/CD Pipeline Using Jenkins, Maven, Tomcat, and AWS S3

## Project Overview

Implemented an end-to-end CI/CD pipeline for deploying a Java-based web application called **DevOps Training Portal - Batch 009** using Jenkins, Maven, Apache Tomcat, and AWS S3.

The project focused on automating the build, deployment, and artifact management process in a Linux-based AWS environment.

![](/ci-cd-pipeline-using-jenkins-maven-tomcatserver-s3-bucket/images/architecture-diagram.png)

## Technologies Used

| Technology      | Purpose                |
| --------------- | ---------------------- |
| Jenkins         | CI/CD Automation       |
| Maven           | Build Automation       |
| Apache Tomcat   | Application Server     |
| AWS S3          | Artifact Storage       |

## Workflow

1. Source code pulled from GitHub repository
2. Jenkins triggered Maven build process
3. WAR artifact generated inside `target/`
4. Jenkins uploaded WAR artifact to AWS S3 bucket
5. S3 versioning maintained multiple build artifacts
6. Jenkins deployed WAR file to Apache Tomcat

```bash
GitHub
   ↓
Jenkins
   ↓
Maven Build
   ↓
WAR Artifact Generated
   ↓
Upload WAR Artifact to S3
   ↓
Deploy WAR Artifact to Tomcat
```

## Step by Step Implementation

### Step 1: Install Jenkins

```bash
sudo dnf update -y

sudo dnf install java-21-amazon-corretto-devel -y

sudo wget -O /etc/yum.repos.d/jenkins.repo \
https://pkg.jenkins.io/redhat-stable/jenkins.repo

sudo rpm --import https://pkg.jenkins.io/redhat-stable/jenkins.io-2023.key

sudo dnf install jenkins -y

sudo systemctl daemon-reload

sudo systemctl enable jenkins

sudo systemctl start jenkins

sudo systemctl status jenkins
```

### Step 2: Install Git and Maven

```bash
sudo yum install git maven -y
```
### Step 3: Install Apache Tomcat

```bash
wget https://dlcdn.apache.org/tomcat/tomcat-9/v9.0.113/bin/apache-tomcat-9.0.113.tar.gz
tar -xvf apache-tomcat-9.0.113.tar.gz
cd apache-tomcat-9.0.113/bin
./startup.sh
```
### Step 4: Change Tomcat Port from 8080 to 9090

#### Update Port
```bash
sudo sed -i 's/port="8080"/port="9090"/' \
apache-tomcat-9.0.113/conf/server.xml
```

#### Restart Tomcat
```bash
cd apache-tomcat-9.0.113/bin

./shutdown.sh

./startup.sh
```

### Step 5: Configure AWS Security Group

Add inbound rules for:

| Port | Purpose |
| ---- | ------- |
| 22   | SSH     |
| 8080 | Jenkins |
| 9090 | Tomcat  |

### Step 6: Configure Tomcat Manager Access

#### Update manager/context.xml

```bash
echo '<Context antiResourceLocking="false" privileged="true">
  <Valve className="org.apache.catalina.valves.RemoteAddrValve"
         allow="^.*$" />
</Context>' | sudo tee apache-tomcat-9.0.113/webapps/manager/META-INF/context.xml
```

#### Update host-manager/context.xml
```bash
echo '<Context antiResourceLocking="false" privileged="true">
  <Valve className="org.apache.catalina.valves.RemoteAddrValve"
         allow="^.*$" />
</Context>' | sudo tee apache-tomcat-9.0.113/webapps/host-manager/META-INF/context.xml
```

#### Step 7: Configure Tomcat Users

```bash
echo '<?xml version="1.0" encoding="UTF-8"?>
<tomcat-users xmlns="http://tomcat.apache.org/xml"
              xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
              xsi:schemaLocation="http://tomcat.apache.org/xml tomcat-users.xsd"
              version="1.0">

  <role rolename="manager-gui"/>
  <role rolename="admin-gui"/>
  <role rolename="manager-script"/>

  <user username="tomcat" password="tomcat" roles="manager-gui"/>

  <user username="admin" password="admin"
        roles="manager-gui,admin-gui,manager-script"/>

</tomcat-users>' | sudo tee apache-tomcat-9.0.113/conf/tomcat-users.xml
```
#### Restart Tomcat

```bash
cd apache-tomcat-9.0.113/bin

./shutdown.sh

./startup.sh
```
**Note:**  To allow Jenkins to deploy applications into Tomcat, we must configure: **manager-script** role inside **tomcat-users.xml**.

## Using pipeline

```bash
pipeline {

    agent any

    environment {

        BUCKET_NAME = "ci-cd-project-5-24-2026"

        TOMCAT_URL = "http://<EC2-PUBLIC-IP>:9090"

        TOMCAT_CREDENTIALS = credentials('tomcat-creds')

    }

    tools {

        maven 'Maven'

        jdk 'JDK21'

    }

    stages {

        stage('Checkout Source') {

            steps {

                git branch: 'main',
                url: 'https://github.com/KastroVKiran/tomcat-app-kastro.git'

            }

        }

        stage('Build Application') {

            steps {

                sh 'mvn clean package'

            }

        }

        stage('Archive Artifact') {

            steps {

                archiveArtifacts artifacts: 'target/*.war'

            }

        }

        stage('Upload WAR to S3') {

            steps {

                sh """

                aws s3 cp target/*.war \
                s3://${BUCKET_NAME}/artifacts/

                """

            }

        }

        stage('Deploy to Tomcat') {

            steps {

                deploy adapters: [

                    tomcat9(

                        credentialsId: 'tomcat-creds',

                        path: '',

                        url: "${TOMCAT_URL}"

                    )

                ],

                contextPath: null,

                war: 'target/*.war'

            }

        }

    }

    post {

        success {

            echo "Deployment Successful"

        }

        failure {

            echo "Pipeline Failed"

        }

        always {

            cleanWs()

        }

    }

}
```
