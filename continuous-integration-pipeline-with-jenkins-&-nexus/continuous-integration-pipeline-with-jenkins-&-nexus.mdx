## Continuous Integration Pipeline using Jenkins, Maven, and Nexus

## Project Overview

This project demonstrates an automated Continuous Integration (CI) pipeline for a Java Spring Boot application using Jenkins, Maven, and Nexus Repository.

The pipeline automates the complete software build process starting from source code checkout to artifact storage in Nexus Repository. Developers push the application code to a GitHub repository, after which Jenkins automatically triggers the pipeline.

## Features

- Automated CI pipeline using Jenkins
- Maven-based build and dependency management
- Automated unit testing
- JAR artifact generation
- Artifact storage in Nexus Repository
- Snapshot and Release repository management
- Version-controlled build artifacts

## Architecture Diagram

![](/continuous-integration-pipeline-with-jenkins-&-nexus/images/architecture.png)
### 1. Git Checkout
Jenkins pulls the latest source code from the GitHub repository.

### 2. Compile
Maven compiles the Java Spring Boot application source code.

### 3. Test
JUnit test cases are executed automatically during the pipeline.

### 4. Build
Maven packages the application and generates the JAR artifact.

### 5. Store Artifact
The generated artifact is uploaded and stored in Nexus Repository for future usage.

## Step by Step Implementation

### Step-1. Install Jenkins on an Ubuntu EC2 instance.

```bash
sudo apt update
sudo apt install openjdk-21-jre-headless -y
or 
sudo apt install openjdk-21-jdk -y
sudo wget -O /usr/share/keyrings/jenkins-keyring.asc \
  https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key
echo "deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc]" \
  https://pkg.jenkins.io/debian-stable binary/ | sudo tee \
  /etc/apt/sources.list.d/jenkins.list > /dev/null
sudo apt-get update
sudo apt-get install jenkins -y
```
### Step-2. Install Maven , Git, Docker and using docker install Nexus Repository.

```bash
sudo apt install maven -y
sudo apt install git -y
# Add Docker's official GPG key:
sudo apt update
sudo apt install ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

# Add the repository to Apt sources:
sudo tee /etc/apt/sources.list.d/docker.sources <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/docker.asc
EOF

sudo apt update

# install docker packages
sudo apt install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl status docker
sudo systemctl start docker
sudo docker run hello-world # to check if docker is installed successfully
sudo usermod -aG docker jenkins # to add jenkins user to docker group which is used to run docker commands without sudo
newgrp docker # to apply the changes to the current shell
```

Install Nexus Repository using docker:

```bash
docker run -d --name nexus3 -p 8081:8081 sonatype/nexus3
docker ps # to check if nexus is running
docker logs nexus3 # to check if nexus is running successfully
```

### Step-3. Add Ports to Security Group of the EC2 instance.

### Step-4. Install Required Jenkins Plugins

```bash
Manage Jenkins → Plugins → Available Plugins

Search and install the following plugins:
- Config File Provider
- Maven Integration
- Eclipse Temurin Installer
- Pipeline Stage View
- Pipeline Maven Integration
```
## Config File Provider Plugin
Used to securely manage configuration files like `settings.xml` inside Jenkins. It helps avoid storing sensitive configuration files in GitHub.

---

## Maven Integration Plugin
Used to integrate Maven builds with Jenkins pipelines. It helps automate compile, test, package, and deploy stages.

---

## Eclipse Temurin Installer Plugin
Used to automatically install and manage different Java versions in Jenkins. It helps projects use required JDK versions easily.

---

## Pipeline Stage View Plugin
Used to visualize Jenkins pipeline stages graphically. It helps monitor pipeline progress and identify failed stages quickly.

---

## Pipeline Maven Integration Plugin
Used to provide advanced Maven support inside Jenkins pipelines using `withMaven`. It automatically publishes reports and manages Maven execution.

### Step-8. Create a Jenkins Pipeline Job

```bash
Jenkins Dashboard -> New Item -> Pipeline -> Name the job -> OK
```

Pipeline Script:

```groovy
pipeline {
    agent any
    tools {
        maven 'maven3'
    }
    stages {
        stage('Git Checkout') {
            steps {
                git branch: 'main', url: 'https://github.com/KastroVKiran/Nexus-Demo.git'
            }
        }
        stage('Compile') {
            steps {
                sh 'mvn compile'
            }
        }
        stage('Tests') {
            steps {
                sh 'mvn test'
            }
        }
        stage('Build') {
            steps {
                sh 'mvn package'
            }
        }
        stage('Deploy Artifacts') {
            steps {
                withMaven(globalMavenSettingsConfig: 'settings.xml', jdk: 'jdk17', maven: 'maven3', mavenSettingsConfig: '', traceability: true) {
                    sh 'mvn deploy'
                }
            }
        }
    }
}
```

# Conclusion

This project successfully demonstrates the implementation of a Continuous Integration (CI) pipeline using Jenkins, Maven, and Nexus Repository for a Java Spring Boot application.

The pipeline automates the complete build process including source code checkout, compilation, testing, packaging, and artifact storage. By integrating Jenkins with Maven and Nexus Repository, the project achieves faster build automation, centralized artifact management, version control, and reduced manual effort.
