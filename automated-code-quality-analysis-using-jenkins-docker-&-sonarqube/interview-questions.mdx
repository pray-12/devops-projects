# DevOps CI/CD Project Interview Explanation

---

# Project Introduction

I worked on an end-to-end DevOps CI/CD automation project using Jenkins, SonarQube, Docker, and Spring Boot application.

The main goal of the project was to automate:
- Build process
- Code quality analysis
- Docker image creation
- Deployment process

The application used in the project was a Spotify Spring Boot application developed using Java 21.

# How to Explain the Project in Interview

I created a CI/CD pipeline where Jenkins automatically pulls source code from GitHub whenever changes are pushed.

After pulling the code:
- Maven builds the application and generates a JAR file.
- SonarQube performs static code analysis to identify bugs, vulnerabilities, and code smells.
- Docker packages the application into a container image.
- The Docker image is pushed to DockerHub.
- Finally, Jenkins deploys the latest Docker container automatically on the server.

This automation reduced manual work and improved deployment speed and consistency.

# Important Jenkins Pipeline Stages

## 1. Git Checkout

Jenkins pulls latest code from GitHub repository.

```groovy
git branch: 'main',
url: 'YOUR_GITHUB_REPO_URL'
```

---

## 2. Maven Build

Application is compiled and packaged.

```bash
mvn clean package
```

Generated artifact:

```text
target/spotify-app.jar
```

---

## 3. SonarQube Analysis

Code quality analysis is performed.

```bash
sonar-scanner
```

Checks:
- Bugs
- Vulnerabilities
- Code smells
- Security issues

---

## 4. Docker Build

Docker image is created using Dockerfile.

```bash
docker build -t pray12/spotify-app:latest .
```

---

## 5. Docker Push

Docker image is pushed to DockerHub.

```bash
docker push pray12/spotify-app:latest
```

---

## 6. Docker Deployment

Docker container is deployed.

```bash
docker run -d -p 5555:5555 pray12/spotify-app:latest
```

---

# Dockerfile Used in the Project

```dockerfile
FROM eclipse-temurin:21-jre-jammy

WORKDIR /app

COPY target/*.jar app.jar

EXPOSE 5555

ENTRYPOINT ["java", "-jar", "app.jar"]
```

# Interview Questions and Answers

## 1. Explain Your Project.

This is an end-to-end CI/CD automation project where Jenkins automates the build, code analysis, Docker image creation, Docker image push, and deployment process.

The pipeline integrates GitHub, Maven, SonarQube, Docker, and DockerHub to automate software delivery.

---

## 2. Why Did You Use Jenkins?

Jenkins is used for:
- Continuous Integration
- Continuous Delivery
- Build automation
- Deployment automation
- Pipeline orchestration

It reduces manual effort and speeds up software delivery.

---

## 3. What Is CI/CD?

### CI (Continuous Integration)

Automatically integrates and tests code changes.

### CD (Continuous Delivery/Deployment)

Automatically deploys applications after successful build and testing.

---

## 4. Why Did You Use SonarQube?

SonarQube is used for static code analysis.

It detects:
- Bugs
- Security vulnerabilities
- Code smells
- Duplicate code

It improves code quality before deployment.

---

## 5. What Is Docker?

Docker is a containerization platform used to package application code and dependencies into containers.

Benefits:
- Lightweight
- Portable
- Fast deployment
- Environment consistency

---

## 6. Difference Between Docker Image and Container?

| Docker Image | Docker Container |
|---|---|
| Blueprint/template | Running instance |
| Static | Dynamic |
| Created using docker build | Created using docker run |

---

## 7. Why Did You Push Image to DockerHub?

DockerHub is used as a centralized image repository.

Purpose:
- Store images centrally
- Deploy from any server
- Maintain image versions

---

## 8. Why Did You Use Java 21?

Java 21 is an LTS version.

Benefits:
- Better performance
- Long-term support
- Security updates
- Modern features

---

## 9. Why Did You Use `--no-cache` During Docker Build?

```bash
docker build --no-cache
```

Purpose:
- Avoid Docker layer caching
- Ensure latest code changes are included in image

---

## 10. What Is the Use of `withSonarQubeEnv`?

It injects:
- SonarQube URL
- Authentication token
- Environment variables

securely into Jenkins pipeline.

---

## 11. Why Add Jenkins User to Docker Group?

To allow Jenkins to execute Docker commands without sudo.

Command:

```bash
sudo usermod -aG docker jenkins
```

---

## 12. What Problems Did You Face During the Project?

### Docker Permission Issue

Error:

```text
permission denied while trying to connect to docker.sock
```

Fix:

```bash
sudo usermod -aG docker jenkins
```

---

### SonarQube Connectivity Issue

Fix:

Used:

```text
http://localhost:9000
```

instead of public IP because Jenkins and SonarQube were running on same server.

---

### DockerHub Authentication Failure

Fix:

Used DockerHub access token instead of account password.
