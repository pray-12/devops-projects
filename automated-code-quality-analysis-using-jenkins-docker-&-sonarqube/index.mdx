# Automated Code Quality Analysis using Jenkins, Docker and SonarQube

## Project Overview

This project is a real-time DevOps CI/CD implementation designed to automate the complete software build, code quality analysis, containerization, and deployment process using Jenkins, SonarQube, Maven, and Docker.

The primary objective of this project is to integrate automated code quality checks into the CI/CD pipeline and deploy the application inside a Docker container for easy portability and consistency across environments.

The pipeline starts when the source code is pulled from a GitHub repository through Jenkins. Maven is then used to compile, test, and package the Java application. During the pipeline execution, SonarQube performs static code analysis to identify bugs, vulnerabilities, code smells, duplications, and maintainability issues in the application code.

After successful code quality validation, Jenkins builds a Docker image for the application and pushes the image to Docker Hub. Finally, the Docker container is launched, allowing users to access the deployed application through the browser.

This project demonstrates real-world DevOps practices such as:

- Continuous Integration (CI)
- Automated Build and Testing
- Static Code Analysis
- Continuous Delivery (CD)
- Containerization using Docker
- Pipeline Automation using Jenkins
- Quality Gate Integration with SonarQube

## Workflow of the Project

1. Developer pushes code to GitHub.
2. Jenkins pipeline gets triggered automatically.
3. Maven compiles and tests the application.
4. SonarQube performs code quality analysis.
5. Maven packages the application.
6. Jenkins builds the Docker image.
7. Docker image is pushed to Docker Hub.
8. Docker container is launched.
9. Application becomes accessible through browser.

### Step 10: Define Jenkins Pipeline Script
```groovy
pipeline {
    agent any
    
    tools{
        maven 'maven3'
        jdk 'jdk21'
    }
    
    environment {
        SCANNER_HOME = tool 'sonar-scanner'
        DOCKERHUB_USERNAME ='pray12'
        DOCKER_IMAGE = "${DOCKERHUB_USERNAME}/spotify-app:latest"
    }
    stages {
        stage('git checkout'){
            steps {
                git branch: 'main', url: 'https://github.com/pray-12/SonarQube-Project-Kastro.git'
            }
        }
        stage ('compile'){
            steps {
                sh "mvn compile"
            }
        }
        stage ('test'){
            steps {
                sh "mvn test"
            }
        }
        stage('SonarQube Analysis') {
            steps {
                withSonarQubeEnv('sonar-server') {
                    sh '''
                    $SCANNER_HOME/bin/sonar-scanner \
                    -Dsonar.projectName=PreethySonar \
                    -Dsonar.projectKey=PreethyKey \
                    -Dsonar.sources=. \
                    -Dsonar.java.binaries=target
                    '''
                }
            }
        }
        stage('build'){
           steps{
                sh 'mvn package'
            }
        }
        stage('Docker Build') {
            steps {
                script {
                    // Building Docker Image
                    sh "docker build -t $DOCKER_IMAGE ."
                }
            }
        }
        stage('Docker Push to DockerHub') {
            steps {
                script {
                    withCredentials([usernamePassword(credentialsId: 'dockerhub-credentials', usernameVariable: 'DOCKERHUB_USERNAME', passwordVariable: 'DOCKERHUB_PASSWORD')]) {
                        echo "Docker Hub Username: ${DOCKERHUB_USERNAME}"  // Check the username
                        echo "Docker Image: ${DOCKER_IMAGE}"  // Check the image
                        sh "docker login -u ${DOCKERHUB_USERNAME} -p ${DOCKERHUB_PASSWORD}"
                        sh "docker push $DOCKER_IMAGE"
                    }
                }
            }
        }

        stage('Run Docker Container') {
            steps {
                script {
                    // Stop any existing container with the same name
                    sh "docker stop spotify-app || true && docker rm spotify-app || true"
                    
                    // Running the container
                    sh "docker run -d --name spotify-app -p 5555:5555 $DOCKER_IMAGE"
                }
            }
        }
    }
}
```
The docker file used in the project is as follows:
```dockerfile
FROM openjdk:17-jdk-slim
WORKDIR /app
COPY target/spotify-app-1.0.0.jar /app/spotify-app.jar
EXPOSE 5555
ENTRYPOINT ["java", "-jar", "spotify-app.jar"]
```
